## 1. 网站一句话定位

这是一个个人作品集网站首页，但它不想表达成传统意义上的“个人简历站”。

它的核心定位是：

> TPY / AI ART LAB：一个用 AI 探索艺术边界，也用工具重建自我秩序的个人数字实验室。

它更像一个“AI 艺术实验室入口”或“数字雕塑展厅首屏”，而不是普通的个人介绍页。

网站目前的首屏承担三个任务：

1. 建立强烈的个人视觉符号：TPY / AI ART LAB。
2. 传达创作方向：AI、艺术、工具、秩序、实验。
3. 让访客通过交互进入一种“系统被激活”的状态。

## 2. 核心理念

这个网站的理念可以拆成四层。

### 2.1 AI 艺术实验室

网站不是把 AI 当成普通工具介绍，而是把 AI 当成一个创作系统、感知系统和实验空间。

它应该表达：

- 用 AI 做艺术探索。
- 用 AI 生成、改造、组织视觉材料。
- 用 AI 作为思考、写作、设计、编码的协作者。
- 把个人创作过程展示成一个持续运行的实验室。

这里的“LAB”不是一个装饰词，而是整个网站的结构母题。

后续所有模块都应该像实验室里的不同区域：

- About 可以是身份档案或系统说明。
- Tools Lab 可以是工具实验区。
- Projects 可以是实验成果陈列区。
- Resume 可以是能力档案。
- Contact 可以是连接端口。

### 2.2 数字雕塑展厅

当前首页右侧使用一张大雕塑图作为主视觉。

雕塑不是单纯装饰，它代表：

- 古典艺术和 AI 图像之间的碰撞。
- 人类审美传统与数字生成系统之间的连接。
- “人的秩序”与“机器的流动数据”之间的张力。

雕塑默认是黑灰白冷色状态。

当鼠标靠近雕塑时，红色粒子、HUD、光标和阴影被激活，意味着：

- 系统被唤醒。
- 观众进入实验范围。
- 原本静态的艺术对象开始响应人的存在。

### 2.3 用工具重建秩序

网站副标题是：

> 用 AI 探索艺术的边界，也用工具重建自己的秩序。

这句话是网站的价值观核心。

它不是只讲“我会做 AI 艺术”，而是表达一种更完整的个人方向：

- 用 AI 进行视觉和内容实验。
- 用工具系统化自己的工作、知识、创作和生活。
- 在混乱的信息时代里，主动建立自己的秩序。

因此后续页面不应该只是列作品，而应该展示：

- 我如何思考。
- 我如何搭建工具。
- 我如何组织项目。
- 我如何把 AI 变成个人系统的一部分。

### 2.4 黑白冷静，红色激活

当前网站有一个明确的色彩逻辑：

- 黑、灰、白是基础状态。
- 红色是激活状态。

红色不应该被大量铺满，也不应该变成普通品牌主色。

红色代表：

- 系统启动。
- 交互发生。
- 数据流开始。
- 观众靠近。
- 某个模块进入可操作状态。

也就是说，红色应该像“信号”，而不是“背景”。

## 4. 当前交互设计

### 4.1 鼠标光点

页面内有一个跟随鼠标移动的光点。

默认状态：

- 白色。
- 半透明。
- 有轻微发光。

鼠标靠近雕塑区域时：

- 光点变成红色发光圆环。
- 发光增强。
- 圆环感更明显。

这个光点不是普通 cursor 替代品，而是像一个“扫描探针”。

### 4.2 雕塑 hover active 状态

当鼠标进入或靠近右侧雕塑区域时，页面进入 active 状态。

active 状态下：

- 红色粒子数量增加。
- HUD 卡片边框变红。
- 雕塑出现红色 drop-shadow。
- 鼠标光点变成红色发光圆环。
- 背景右侧红色光晕增强。

这代表“实验室系统识别到观众接近雕塑对象”。

### 4.3 雕塑视差

雕塑会根据鼠标相对屏幕中心的位置产生轻微变化。

效果包括：

- translateX。
- translateY。
- rotateX。
- rotateY。

幅度很小，不追求夸张晃动。

设计原则：

- 让雕塑“活起来”。
- 不让人觉得眩晕。
- 不要破坏极简和高级感。

### 4.4 粒子系统

背景有少量白色粒子。

active 时，鼠标附近和雕塑区域出现红色粒子流。

性能策略：

- 使用 Canvas 2D。
- 使用 requestAnimationFrame。
- active 状态和鼠标位置放在 ref 中，避免每一帧触发 React 重渲染。
- 移动端关闭复杂红色粒子。
- prefers-reduced-motion 下也降低粒子复杂度。

### 4.5 ENTER LAB 转场

点击 ENTER LAB 后：

1. 页面出现全屏黑色覆盖层。
2. 红色扫描线穿过页面。
3. 显示文字：

```text
SYSTEM ACTIVATED
```

4. 约 800ms 后自动滚动到第二屏。
5. 覆盖层消失。

这个交互的含义：

- 不是普通跳转。
- 是“进入实验室系统”。
- 用户不是点开页面，而是激活系统。

### 6.2 src/App.tsx

负责整体页面结构：

- 渲染 Hero。
- 渲染第二屏占位模块。

第二屏内容目前只是占位，后续可以继续扩展。

### 6.3 src/components/Hero.tsx

这是当前最核心的组件。

它负责：

- Hero 首屏布局。
- 鼠标位置。
- 雕塑 active 状态。
- 雕塑视差。
- ENTER LAB 转场。
- 组合 Navbar、CursorGlow、ParticleField、HudCard。

重要状态：

```ts
mouse
nearSculpture
transitioning
```

重要性能设计：

- 鼠标移动通过 requestAnimationFrame 节流。
- 雕塑区域位置不是每次鼠标移动都重新计算。
- 使用 ResizeObserver 缓存雕塑区域 bounds。
- Framer Motion spring 用于平滑位移和旋转。

### 6.4 src/components/CursorGlow.tsx

负责鼠标光点。

它接收：

```ts
mouse
active
```

默认白色发光。

active 时变成红色发光圆环。

### 6.5 src/components/ParticleField.tsx

负责全屏粒子。

它接收：

```ts
active
mouse
```

默认绘制白色粒子。

active 时绘制红色粒子流。

移动端和低动态偏好下会降低复杂度。

### 6.6 src/components/HudCard.tsx

负责 HUD 数据卡片。

它接收：

```ts
eyebrow
value
active
delay
className
```

active 时：

- 边框变红。
- 小圆点变红。
- 数据条部分变红。
- 阴影增强。

### 6.7 src/components/Navbar.tsx

负责顶部导航。

当前导航没有真实路由逻辑，大部分链接指向第二屏。

后续可以给每个 section 加独立 id。

### 6.8 src/styles.css

负责全局样式和一些关键视觉效果。

包括：

- 全局字体。
- 背景。
- 网格噪声。
- 扫描线 mask。
- 雕塑默认和 active filter。
- 鼠标光点默认和 active 样式。

关键类：

```css
.grid-noise
.scanline-mask
.image-desaturate
.image-active
.cursor-glow-idle
.cursor-glow-active
```

## 9. 后续可以添加的内容模块

下面是适合这个网站继续扩展的方向。

### 9.1 About / 关于我

不要写成普通自我介绍。

建议写成：

```text
SYSTEM PROFILE
```

可以包含：

- TPY 是谁。
- 为什么做 AI 艺术。
- 为什么关注工具和秩序。
- 当前创作方向。
- 当前技能组合。
- 当前正在探索的问题。

表达方式可以像系统档案：

```text
IDENTITY
AI ARTIST / TOOL BUILDER / SYSTEM DESIGNER

CURRENT FOCUS
AI image workflow / personal knowledge system / creative automation
```

### 9.2 Tools Lab / 工具实验室

这是很重要的模块。

它可以展示：

- 自己搭建的 AI 工作流。
- Obsidian 知识整理系统。
- 内容生产工具。
- 图像生成流程。
- 自动化脚本。
- 个人效率系统。

建议做成“实验台”而不是“工具列表”。

每个工具可以有：

- 工具名。
- 解决的问题。
- 输入。
- 输出。
- 使用场景。
- 当前状态。

示例结构：

```text
TOOL EXPERIMENT 01
Obsidian Knowledge Sorting System

Problem:
大量对话和材料难以沉淀。

Input:
原始对话、文章草稿、灵感碎片。

Output:
主题笔记、索引、双链知识结构。
```

### 9.3 Projects / 项目

项目不要做成普通作品卡片。

建议做成“实验成果陈列”。

每个项目可以包含：

- 项目编号。
- 项目名称。
- 类型。
- 核心问题。
- 使用工具。
- 最终输出。
- 视觉截图。
- 实验状态。

示例：

```text
PROJECT 03
AI Sculpture Visual System

Type:
Generative visual experiment

Question:
古典雕塑如何被数字粒子和数据界面重新解释？

Tools:
ChatGPT / Image generation / React / Canvas
```

### 9.4 Resume / 简历

简历也不要太传统。

可以做成：

```text
CAPABILITY MATRIX
```

分成：

- AI 创作能力。
- 前端实现能力。
- 内容策划能力。
- 工具搭建能力。
- 项目管理能力。
- 视觉审美能力。

也可以保留一个下载传统简历的按钮，但页面展示方式应该更像能力矩阵。

### 9.5 Contact / 联系我

联系页可以叫：

```text
CONNECT PORT
```

可以包含：

- 邮箱。
- 微信。
- GitHub 或作品平台。
- 小红书 / 公众号 / 即刻等内容平台。
- 合作方向。

建议文案：

```text
If your project needs AI visual experiments, creative tooling, or a system to organize chaos, open a channel.
```

中文方向：

```text
如果你正在寻找 AI 视觉实验、创作工具搭建，或一个能把混乱信息整理成系统的人，可以联系我。
```

## 11. 内容风格建议

### 11.1 中文文案语气

应该是：

- 克制。
- 有质感。
- 有一点系统感。
- 不油腻。
- 不营销。
- 不堆 AI 流行词。

适合的表达：

```text
我用 AI 扩展视觉想象，也用工具重新组织工作和生活。
```

```text
这个网站不是作品陈列柜，而是一个仍在运行的实验室。
```

```text
每一个项目都是一次对秩序、图像和工具关系的测试。
```

不适合的表达：

```text
我是一个热爱生活、积极向上的设计师。
```

```text
欢迎来到我的个人主页，我擅长前端开发和 AI 绘画。
```

```text
用科技赋能未来，用创新点亮梦想。
```

这些太普通或太像模板。

### 11.2 英文标签风格

适合使用短标签：

```text
SYSTEM PROFILE
POINT CLOUD
DATA FLOW
```

## 12. 技术扩展建议

### 12.1 短期

短期保持单页结构即可。

可以继续扩展：

- About section。
- Tools Lab section。
- Projects section。
- Resume section。
- Contact section。

不需要马上引入路由。

### 12.2 中期

如果项目多起来，可以加入：

- React Router。
- 项目详情页。
- JSON 或 TS 文件存储项目数据。
- 简单 markdown 内容渲染。

例如：

```text
src/data/projects.ts
src/data/tools.ts
src/pages/ProjectDetail.tsx
```

### 12.3 长期

如果网站变成长期内容系统，再考虑：

- CMS。
- 数据库。
- 后台管理。
- 内容搜索。
- 多语言。

但现在不建议做这些。

原因：

- 当前重点是个人网站视觉和内容表达。
- 过早加入后端会拖慢迭代。
- 目前内容量还不需要数据库。

## 14. 重要约束

后续所有方案都要遵守这些约束。

### 14.1 不要变成普通简历站

不能只是：

```text
个人介绍
技能列表
项目列表
工作经历
联系方式
```

而应该是：

```text
系统档案
工具实验室
实验成果
能力矩阵
连接端口
```

### 14.2 不要过早复杂化技术

暂时不要加：

- 后端。
- 数据库。
- 登录。
- 用户系统。
- 复杂 CMS。

### 14.3 不要破坏现有视觉语言

后续新增模块应继续使用：

- 黑底。
- 细线。
- 白字。
- 红色激活。
- HUD。
- 粒子或扫描元素。
- 克制的动效。

### 14.4 红色只能作为激活信号

红色用于：

- hover。
- active。
- CTA。
- 当前状态。
- 系统警示。
- 扫描线。

不要把红色用于：

- 大面积背景。
- 普通装饰块。
- 所有标题。
- 所有按钮。

### 14.5 动效要轻

动效应服务于“系统响应”，不是炫技。

保持：

- 轻微。
- 快速。
- 可感知。
- 不打扰阅读。
